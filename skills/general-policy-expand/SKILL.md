# General Policy Expand

按下面规则处理政策类搜索扩词，目标是“实际、稳、不跑偏”。

## 核心模式识别

先判断是不是政策搜索：
- 含“政策”
- 或含“实施意见 / 管理办法 / 若干措施 / 发展规划 / 通知 / 公告 / 工作方案 / 建设方案”等政策标题型词

再分三类：

### 1. 单省政策模式
触发：出现具体省份/直辖市/自治区，且不是“全国”。

例如：
- 河北的政策
- 河北省博物馆相关政策
- 2026年湖北农业政策

规则：
- 最多扩 8 个词
- 每个词都必须带该省份名
- 每个词都必须绑定该省份官网域名锚点
- 扩词优先用真实文件标题型词，不要发散到太远的话题

### 2. 全国政策模式
触发：出现“全国 / 全国范围 / 各省 / 全国各地 / 全国政策”。

规则：
- 按 31 个省级地区展开
- 每个省固定 2 个扩词
- 每个词都必须带对应省份名
- 每个词都必须绑定对应省级官网域名锚点
- 不能把“全国”直接当成一个搜索站点；要拆到各省去执行

### 3. 普通搜索模式
触发：不是政策搜索。

规则：
- 不套用本技能的省份域名强约束
- 不强行扩到 8 个
- 普通搜索保持自然、轻量

## 主题清洗

在识别主题前，先清掉口语残片：
- 帮我搜索一下
- 帮我搜一下
- 搜索一下
- 搜一下
- 搜索
- 搜
- 相关内容
- 政策相关内容
- 相关
- 内容
- 一下
- 的
- 方面
- 领域

例如：
- “帮我搜索一下河北的政策” → 泛政策（无明确主题）
- “搜索一下湖北博物馆相关政策” → 主题 = 博物馆政策
- “河北农业政策相关内容” → 主题 = 农业政策

不要生成这种错词：
- 搜索一下的政策
- 博物馆内容政策
- 河北相关政策内容政策

## 扩词原则

### A. 泛政策（未给明确主题）
单省时，不硬凑大而空的一级分类裸词；优先使用更可检索、像政策标题的方向词。

默认 8 个核心方向：
1. 发展规划
2. 实施意见
3. 若干措施
4. 管理办法
5. 工作通知
6. 建设方案
7. 申报指南
8. 专项政策

生成时要与省份结合，例如：
- 2026年 河北省发展规划 site:hebei.gov.cn
- 2026年 河北省实施意见 site:hebei.gov.cn

### B. 明确主题政策
如果用户提供主题，例如：博物馆、文旅、农业、新能源、教育、科技、人才、产业、交通、医疗、招商、环保。

优先扩成“主题 + 文件标题型词”，不要过度发散。

示例：博物馆政策
- 博物馆发展规划
- 博物馆实施意见
- 博物馆管理办法
- 博物馆建设方案
- 博物馆扶持政策
- 文物保护实施意见
- 公共文化服务若干措施
- 博物馆工作通知

示例：文旅政策
- 文化旅游发展规划
- 文化旅游实施意见
- 文旅产业扶持政策
- 文旅项目建设方案
- 旅游管理办法
- 公共文化服务若干措施
- 文旅工作通知
- 旅游专项政策

要求：
- 总数仍不超过 8 个
- 主题相近可以替换成贴近公文口径的近义表达，但不能跑偏到无关赛道

### C. 全国主题政策
按每省固定 2 个词：
- 主题 + 发展规划
- 主题 + 实施意见

如果没有明确主题，则按泛政策两条：
- 发展规划
- 实施意见

## 域名锚点

扩词结果必须绑定以下省级政府官网域名：

读取 `references/province_domains.md`。

输出时必须显式保留：
- 省份
- 域名锚点
- 扩展方向
- 搜索关键词

## 输出格式

优先输出为结构化列表，每项包含：
- 年份
- 地区
- 域名锚点
- 原始主题
- 扩展方向
- 搜索关键词
- 模式（单省 / 全国 / 普通）

搜索关键词规则：
- 单省：`{年份}年 {省份}{扩展方向} site:{域名host}`
- 全国：`{年份}年 {省份}{扩展方向} site:{域名host}`
- 普通：不强制拼 site

注意：
- `site:` 里只放 host，不放 https://
- 例如 `site:hebei.gov.cn`

## 资源

- 域名表：`references/province_domains.md`
- 生成脚本：`scripts/expand_policy_query.py`

需要批量生成时，运行：

```bash
python3 scripts/expand_policy_query.py "帮我搜索一下河北的政策"
python3 scripts/expand_policy_query.py "全国博物馆政策"
```

如果只是对话中回答，不必强行运行脚本；但输出规则必须与脚本一致。
