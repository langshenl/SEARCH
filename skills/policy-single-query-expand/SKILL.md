# Policy Single Query Expand

## Overview

这个技能只做一件事：

把自然语言政策搜索需求清洗并转换成 **唯一一个最终搜索词**，直接交给后续多页模拟点击搜索流程使用。

它不再返回多个候选，不再要求“默认选第 1 条”。

## Rules

### 1. 年份规则

- 如果用户明确给了年份，就使用该年份
- 如果用户没有明确给年份，就默认使用当前年份

### 2. 搜索词清洗规则

需要去掉口语残片，例如：
- 帮我搜索一下
- 帮我搜一下
- 搜索一下
- 搜一下
- 搜索
- 搜
- 一下
- 相关内容
- 相关
- 内容

例如：
- `帮我搜索一下湖北省政策` → `湖北省政策`
- `搜索河北相关政策` → `河北政策`
- `搜索湖北省新能源相关政策` → `湖北省新能源政策`

### 3. 域名规则

- 如果能识别出具体省份，就匹配该省政府域名
- 如果是全国，就统一使用 `site:gov.cn`

### 4. 最终输出规则

单省：
- `年份 + 搜索关键词 + site:对应省份域名`

全国：
- `年份 + 搜索关键词 + site:gov.cn`

### 5. 输出形式

只输出一个最终结果对象，不输出候选列表。

## Scripts

- `scripts/build_single_policy_query.py`
  - 输入自然语言搜索需求
  - 输出唯一最终搜索词及结构化信息

## Hard Rules

- 不生成 CSV
- 不写桌面配置文件夹
- 不返回多候选
- 不执行“默认选第 1 条”，因为结果只有一个
- 输出结果直接供后续多页模拟点击搜索流程使用
