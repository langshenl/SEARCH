# Detail Link Fetcher

## Overview

这个技能作为第六步使用。

它负责：
- 读取 `~/Desktop/处理文件夹/原文链接.md`
- 按 1 秒间隔批量直抓原文链接
- 将固定字段整理成 Excel
- 保存到 `~/Desktop/detail-h5/`

## Fixed Workflow

### 1. 读取原文链接文件

固定输入文件：
- `~/Desktop/处理文件夹/原文链接.md`

### 2. 批量直抓

规则：
- 逐条读取链接
- 串行抓取
- 每条之间间隔 1 秒

### 3. 固定字段输出

输出 Excel 中必须始终包含以下字段名，即使抓不到也要保留空值：

- 政策ID
- 标题
- 正文
- 摘要
- 发文机关
- 联合发文机关
- 发布时间
- 文号
- 政策层级
- 地区
- 所属行业
- 政策类型
- 政策主题
- 关键词
- 适用对象
- 支持方式
- 申报条件
- 申报时间
- 有效期
- 政策状态
- 原始链接
- 附件链接
- 相似政策
- 上位政策/下位政策关系
- 抓取状态
- 正文长度
- 是否检测到附件
- 是否需要浏览器兜底

### 4. 输出目录

固定输出目录：
- `~/Desktop/detail-h5/`

默认输出文件：
- `政策详情抓取结果.xlsx`

## Hard Rules

- 字段名必须完整存在，不允许因为抓不到数据就删列。
- 抓不到的数据留空。
- 本技能优先采用直抓，不默认打开 Safari。
- 本技能应尽量给出“抓取状态 / 正文长度 / 是否检测到附件 / 是否需要浏览器兜底”，用于判断缺失程度。
- 如后续需要浏览器兜底，应在第七步单独处理，不要混进本技能。
