# Policy Search Workflow Memory

按这套固定流程理解和执行政策搜索请求，除非用户明确要求改流程。

## 核心原则

- 用户只提供自然语言需求，不应被要求手动再跑扩词、再填表、再指定执行层。
- 默认直接走本地脚本/主脚本链路，**不允许创建子agent执行**，除非用户明确要求。
- 内部固定流程：
  1. 解析自然语言
  2. 生成扩展临时表
  3. 主脚本读取临时表逐行执行
  4. 每行执行后回写状态
  5. 全部完成后删除临时表（如条件满足）

## 解析规则

- 没写年份 → 默认当前年份
- 没写地区 → 默认湖北省
- 写了“全国” → 全国模式
- 写了具体省份/自治区/直辖市 → 单省模式

## CSV 固定格式（硬规则）

`current-expansion-plan.csv` 必须固定为以下 8 列，顺序也固定：

1. 年份
2. 地区
3. 原始主题
4. 扩展方向
5. 搜索关键词
6. 推荐部门
7. 状态
8. 执行备注

主脚本必须按 **搜索关键词** 字段执行，不允许再漂字段名。

## 范围规则

- 全国模式：按省份展开，**每省固定 3 行**
- 单省模式：通常固定 20 行
- 但如果用户没有提及具体政策主题，只是搜索“某省政策内容 / 某省相关政策内容”，则**不要硬凑 20 条**，直接走 8 大核心领域：
  1. 粮食安全政策
  2. 耕地保护政策
  3. 产业发展政策
  4. 绿色生态政策
  5. 科技装备政策
  6. 经营主体政策
  7. 金融保险政策
  8. 乡村建设政策

这条规则优先级高于“单省固定20行”。

## 扩词质量规则

### 普通主题模式

例如：农业、博物馆、教育、新能源、道路。

- 命中模板 → 用模板扩展
- 未命中模板 → 用通用方向扩展
- 但普通主题扩词必须优先使用更像真实政府文件标题的词，例如：
  - 发展规划
  - 实施意见
  - 若干措施
  - 管理办法
  - 建设方案
  - 专项规划
  - 工作通知
  - 申报指南
- 泛方向词（如提升、保障、监管、创新）可以保留，但应后置，不要默认排在最前面。

### 全量政策模式

- 不能直接使用一级分类裸词作为搜索词，例如：经济政策、科技政策、工业政策、招商投资政策。
- 单省模式应直接生成可检索执行词，或在泛政策内容场景下直接使用 8 个核心方向。

## 主题清洗规则

扩词脚本必须先做主题标准化，清理口语残片，例如：
- 帮我搜索一下 / 帮我搜一下 / 搜索一下 / 搜一下 / 搜索 / 搜
- 相关内容 / 政策相关内容 / 相关 / 内容
- 帮我 / 一下 / 的 / 方面 / 领域

例如：
- `搜索一下广东的博物馆相关内容` → 应标准化为 `博物馆政策`

不能再生成类似：
- `搜索一下博物馆内容政策`

## 主脚本执行规则

当前主脚本：
- `~/Desktop/政策采集系统/policy_harvest_system.py`

执行层必须：
- 严格以配置表里的 **地区** 字段为准
- 通过 `PROVINCE_DOMAINS` 获取地区对应域名锚点
- 不允许再根据原始需求中的“全国”等字样反向改写执行范围
- 如果配置表中的扩展方向已经是可执行方向词，则不得再次展开

## 单行执行性能规则

主脚本单行执行默认采用：
- **轻量 3 轮**
  1. 主站搜索
  2. 部门补搜
  3. 目录补搜

并启用早停：
- 当前行已经召回到结果 → 提前结束本行
- 连续 3 轮都是 0 结果 → 提前结束本行

不要把地市补搜 / 专题补搜当作每行默认必跑项。

## 状态回写规则

### 成功
- `状态 = 执行`
- `执行备注 = 完成，原始结果 X 条；结果文件：xxx.json`

### 失败
- `状态 = 失败`
- `执行备注 = 失败原因`

### 高重复
- `状态 = 跳过重复方向`
- `执行备注 = 结果重叠率过高：xx.xx%`

### 失败原因分层

优先区分：
- 搜索源未召回结果
- 召回后无有效结果

## 目录规则（最新）

### 最外层运行目录

原始数据和最终数据的最外层目录，不再只用年份。
必须使用：
- `年份-时分秒`

例如：
- `2026-201530`

### 原始数据目录

- 单省模式：
  - `~/Desktop/原始数据/{year-hhmmss}/{省份}/`
- 全国模式：
  - `~/Desktop/原始数据/{year-hhmmss}/全国/{省份}/`

重点：
- **全国模式下必须先创建“全国”文件夹，再在里面按省份细分**
- 不允许把全国模式的各省结果直接平铺在最外层运行目录下面

### 最终数据目录

- 单省模式：
  - `~/Desktop/最终数据/{year-hhmmss}/{省份}/`
- 全国模式：
  - `~/Desktop/最终数据/{year-hhmmss}/全国/`

## 路径记忆

- 配置表：`~/Desktop/搜索配置文件夹/current-expansion-plan.csv`
- 搜索脚本原始输出：`~/Desktop/桌面政策文件夹/`
- 主脚本原始汇总目录：`~/Desktop/原始数据/`
- 主脚本最终结果目录：`~/Desktop/最终数据/`

## 一句话总流程

用户给自然语言需求 → 系统解析年份/地区/主题 → 扩词技能按固定 8 列格式生成配置表 → 单省泛政策内容直接走 8 大核心方向，普通主题优先生成文件标题型执行词 → 主脚本按“搜索关键词”执行 → 单行采用轻量 3 轮 + 早停 → 每行回写状态和备注 → 结果写入带“年份-时分秒”的运行目录；全国模式原始数据先入“全国”总目录再按省份细分。
