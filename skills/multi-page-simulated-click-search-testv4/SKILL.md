# Multi-Page Simulated Click Search Test v4

## 概述

政策/官方文件搜索的完整自动化流水线。将自然语言需求转化为结构化政策数据库，支持多省份、多关键词搜索。

**与 v4 的关系：** 脚本与 v4 完全同步，SKILL.md 单独维护。

## 路由判断

| 触发条件 | 适用技能 |
|---------|---------|
| query 含政策/法规/通知/意见等政策词，或科技创新/经济发展等政府政策领域词 | v4（site:gov.cn） |
| 话题/事件/产品/技术等非政策类搜索 | general-search（百度全网） |

**核心原则：不脑补，按 query 原文判断。**

## 目录结构

```
~/Desktop/搜索文件夹/
├── h5/{省份}+{搜索词} site_{域名}_{时间戳}/
│   └── {时间戳}_{搜索词}_page{N}.md    # 每页 HTML 源码
├── 处理文件夹/
│   ├── 搜索结果.xlsx          # 标题/简介/百度链接/原文链接
│   └── 原文链接.md             # 去重后原始链接列表
└── detail-h5/{省份_关键词}_{时间戳}.xlsx  # 政策详情
```

## 单省搜索流程

### 第 1 步：扩词

```bash
python3 scripts/build_single_policy_query.py "2026年湖北省新能源政策"
```

**扩词规则：**
- 年份区间 → `2020-01-01,2026-12-31`；单年份 → `2026-01-01,2026-12-31`；无年份 → 默认当前年
- 省份匹配政府域名：单省 `关键词 site:www.{省}.gov.cn`，全国 `关键词 site:gov.cn`
- **结果数量（可选）**：`至少30条` → 自动换算 `ceil(30/10)=3` 页，上限10页
- 只输出一个唯一结果，不返回候选列表

**输出示例（单省）：**
```json
{
  "年份": "2026-01-01,2026-12-31",
  "地区": "湖北省",
  "域名锚点": "https://www.hubei.gov.cn",
  "清洗后关键词": "新能源政策",
  "最终搜索词": "新能源政策 site:www.hubei.gov.cn",
  "模式": "单省"
}
```

### 第 2 步：Safari 多页抓取

```bash
bash scripts/capture_multi_page_baidu.command "新能源政策 site:www.hubei.gov.cn" 3 "2026-01-01,2026-12-31"
```

**参数：** 搜索词、最大页数（默认3）、年份区间（可选）

**Safari 关键机制：**
- 创建全新 Safari 窗口搜索
- 点击时间筛选下拉框，填写年份区间
- 逐页保存 HTML 源码到 `~/Desktop/搜索文件夹/h5/`
- 完成后关闭窗口，不影响原窗口

### 第 3 步：合并处理

```bash
python3 scripts/process_search_h5_multi.py "新能源政策"
```

**输入：** 搜索词前缀，自动模糊匹配对应 H5 目录
**输出：** `~/Desktop/搜索文件夹/处理文件夹/搜索结果.xlsx` + `原文链接.md`

### 第 4 步：批量直抓详情

```bash
python3 scripts/fetch_detail_links.py "新能源政策"
```

**输入：** `~/Desktop/搜索文件夹/处理文件夹/原文链接.md`
**输出：** `~/Desktop/搜索文件夹/detail-h5/{关键词}_{时间戳}.xlsx`

## 多省搜索流程

### 一键多省

```bash
python3 scripts/multi_province_search.py "湖南湖北新能源"
```

**流程：** 扩词 → 逐省 Safari 抓取 → 合并所有省 H5 → 合并处理 → 详情抓取

**支持两种 H5 目录命名格式：**
- 有省份前缀：`湖北+新能源 site_www.hubei.gov.cn_时间戳/`
- 无省份前缀：`新能源 site_www.hubei.gov.cn_时间戳/`

## 输出字段（9列）

标题、正文、摘要、发文机关、发布时间、原始链接、关键词、类型、地区

**附加信息（内置于抓取逻辑）：**
- 抓取状态、正文长度、是否需要浏览器兜底

## 环境依赖

- macOS + Safari + osascript
- Python 3 + openpyxl

## Hard Rules

- 只走单结果扩词主路径，不混单页逻辑，不返回候选列表
- 页数上限只是上限，抓到哪里算哪里
- 详情抓取先去重再抓
- JS 渲染页面导致正文字数过短 → 标记 `需要浏览器兜底: 是`
