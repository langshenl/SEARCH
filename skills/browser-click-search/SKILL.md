# Browser Click Search

## Overview

这个技能当前只做一件事：

**在用户已经从扩词结果中选定某一条搜索词后，模拟使用 Safari 打开百度，输入该条搜索词，并把当前搜索结果页的完整 H5/HTML 页面源码保存到桌面 `h5` 文件夹。**

不要在这个技能里顺手做：
- 扩词
- 结果提取
- 去重清洗
- 点击多个结果继续深抓
- 政策库整理

先把“打开页面并保存整页源码”这件事单独做好。

## Fixed Workflow

### 1. 接收用户选中的搜索词

只使用用户从扩词结果中明确选中的那一条搜索词。

默认规则：
- 本技能不负责扩词
- 不自动补 `site:`
- 不替用户改写搜索词
- 进入本技能前，应已经完成“扩词 → 返回候选词 → 用户选中一条”这一步

### 2. 打开 Safari

使用 Safari 作为固定浏览器。

目标动作：
- 启动 Safari
- 打开百度
- 在百度搜索框输入用户搜索词
- 进入对应搜索结果页

### 3. 获取当前搜索结果页完整源码

核心目标不是“截图”，而是：

- 获取当前搜索结果页完整 H5 / HTML 页面内容
- 尽量保存渲染后的当前页面源码或等效 HTML 内容

如果需要借助 Safari 开发者工具、页面另存、浏览器导出、自动化脚本等方式实现，可以使用；但最终目标始终是：

**得到当前搜索结果页对应的完整页面文件。**

### 4. 保存到桌面 h5 文件夹

输出目录固定为：
- `~/Desktop/h5/`

要求：
- 如果文件夹不存在，则创建
- 每次任务保存为独立文件
- 文件名应包含：时间戳 + 搜索词的安全文件名

推荐文件名格式：
- `YYYYMMDD_HHMMSS_搜索词.html`

例如：
- `20260331_100300_湖北省政策.html`

### 5. 输出说明

完成后返回给用户：
- 使用的搜索词
- 保存的文件完整路径
- 若失败，明确失败在哪一步（Safari 未打开 / 百度未加载 / 页面源码未取到 / 文件写入失败）

## Hard Rules

- 这个技能当前只负责“打开百度搜索页并保存整页 H5/HTML 文件”。
- 不在本技能中自动扩词。
- 不在本技能中自动抽取标题、链接、摘要。
- 不在本技能中直接做政策清洗。
- 不要把“保存整页 HTML”和“解析搜索结果”混成同一步。

## Output Path

固定输出目录：
- `~/Desktop/h5/`

## Future Extension

后续如果要扩展，再考虑加：
- 从保存的 HTML 中提取搜索结果
- 判断页面是否为安全验证页
- 自动识别真实落地链接
- 作为其他搜索技能的前置抓取层

但当前版本先不要扩出去。
