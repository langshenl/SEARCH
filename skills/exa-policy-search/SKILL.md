# Exa Policy Search

## 输入规范

自然语言输入，支持3种模式：

| 输入模式 | 示例 | 地区字段 |
|----------|------|----------|
| 单省搜索 | `湖北省博物馆` / `河北乡村振兴政策` | 用户输入中识别的省份 |
| 全国搜索 | `全国新能源汽车政策` / `碳中和政策` | 全国 |
| 纯主题搜索 | `乡村振兴` | 尝试识别，未识别写"全国" |

- **默认条数**：20条（用户未指定时）
- **关键词**：直接使用用户输入内容作为搜索词

## 工作流程

### 第1步：解析输入

从自然语言中识别地区和搜索主题：
- 输入含省份名（湖北/河北等）→ 地区=该省份
- 输入含"全国" → 地区=全国
- 其他情况 → 地区=全国，主题=原词

### 第2步：调用 Exa 搜索

用 `web_search_exa` 工具，参数：
```json
{
  "query": "<搜索词>",
  "numResults": 20,
  "type": "auto"
}
```

### 第3步：结构化映射

将 Exa 返回的每条结果映射为 9 个字段：

| 字段 | 来源 |
|------|------|
| 标题 | `title` |
| 正文 | `text` / `content`；无则用 `summary` |
| 摘要 | `summary` / `snippet` |
| 发文机关 | 从标题+摘要+正文中用正则提取机构名；无则留空 |
| 发布时间 | 从日期字段或文本中用正则抽取；无则留空 |
| 原始链接 | `url` |
| 关键词 | 用户原始输入 |
| 类型 | 从标题/正文中识别"通知/办法/意见/方案/公告/政策/网页内容" |
| 地区 | 第1步识别的地区 |

### 第4步：写入结果

1. 将原始结果保存为临时 JSON
2. 执行转换脚本：

```bash
python3 scripts/write_exa_results.py <临时json路径> "<关键词>" "<地区>"
```

脚本自动在 `~/Desktop/exa搜索文件夹/` 下创建时间戳目录，生成：
- `搜索结果.xlsx`（9列）
- `搜索日志.md`

### 第5步：返回用户

返回：
- 写入条数
- Excel 路径
- 输出目录

## 9列字段

标题、正文、摘要、发文机关、发布时间、原始链接、关键词、类型、地区

## 注意

- 若 Exa 返回字段不足，按规则尽量补齐；补不出的字段允许留空
- 不要额外调用其他搜索工具，只用 Exa
