# 多页政策搜索工作流详细步骤

## 完整工作流

### 第1步：输入原始搜索需求

用户输入自然语言需求，例如：
- 搜索湖北省政策
- 搜索河北省博物馆政策
- 搜索广西政策内容

### 第2步：单结果扩词

调用 `policy-single-query-expand` 技能生成唯一搜索词：
- 只生成**一个**最终搜索词
- 不返回候选列表
- 不等待用户选词
- 直接进入下一步

扩词规则：
- 有明确年份用该年份，否则默认当前年份
- 单省：`{年份} {关键词} site:{对应政府域名}`
- 全国：`{年份} {关键词} site:gov.cn`

例如：
- 搜索湖北省政策 → `2026年 湖北省政策 site:www.hubei.gov.cn`
- 搜索湖北省新能源相关政策 → `2026年 湖北省新能源政策 site:www.hubei.gov.cn`
- 搜索全国新能源政策 → `2026年 全国新能源政策 site:gov.cn`

### 第3步：Safari 多页抓取搜索结果

使用桌面脚本：
```
~/Desktop/百度_Safari_H5抓取_多页.command
```

脚本动作：
1. Safari 打开百度
2. 输入搜索词
3. 进入搜索结果页
4. 点击"时间不限"
5. 填入本年的第一天和最后一天
6. 点击"确认"
7. 保存第1页搜索结果 H5/HTML 到 `~/Desktop/h5/`
8. 自动滚动到页码区
9. 点击"下一页"或可用页码
10. 继续保存后续页面 H5
11. 到达目标页数上限或没有下一页时停止
12. **保存完成后关闭当前搜索页面，但保留 Safari 窗口存在**

规则：
- 目标页数是上限，不保证一定抓满
- 如果实际页码不够，就抓到哪里算哪里
- 没有下一页时正常停止，不报错

### 第4步：合并处理多页搜索结果 H5

调用 `search-h5-processor` 处理脚本：
```
~/.openclaw/workspace-search/skills/search-h5-processor/scripts/process_search_h5_multi.py
```

输入：`~/Desktop/h5/` 中同一批次前缀的多页搜索结果文件

输出到：`~/Desktop/处理文件夹/`

固定输出文件：
- `搜索结果.xlsx`
- `原文链接.md`

规则：
- 覆盖上次结果，不做追加
- 合并多页结果后再去重
- `搜索结果.xlsx` 只保留字段：标题、简介、原始百度链接、原文链接
- `原文链接.md` 优先写入真实政府落地链接

### 第5步：结果判断

如果第4步提取结果数为 0：
- **直接返回：`暂未搜索到相关内容`**
- 不继续第6步

如果第4步结果数大于 0：
- **立刻自动进入第6步**
- 不停下来等待额外确认

### 第6步：批量直抓原文链接

调用 `detail-link-fetcher` 脚本：
```
~/.openclaw/workspace-search/skills/detail-link-fetcher/scripts/fetch_detail_links.py
```

输入：`~/Desktop/处理文件夹/原文链接.md`

规则：
- 串行抓取
- 每条间隔 1 秒
- 先对原文链接去重，再按唯一链接抓取
- 优先采用直抓，不默认打开 Safari

输出到：`~/Desktop/detail-h5/`

默认输出文件命名：
`YYYYMMDD_HHMMSS_{搜索关键词去域名}_政策详情抓取结果.xlsx`

### 第7步：政策详情字段

第6步输出的 Excel 必须包含以下字段：
- 政策ID
- 标题
- 正文
- 摘要
- 发文机关
- 联合发文机关
- 发布时间
- 文号
- 政策层级
- 地区
- 所属行业
- 政策类型
- 政策主题
- 关键词
- 适用对象
- 支持方式
- 申报条件
- 申报时间
- 有效期
- 政策状态
- 原始链接
- 附件链接
- 相似政策
- 上位政策/下位政策关系
- 抓取状态
- 正文长度
- 是否检测到附件
- 是否需要浏览器兜底

规则：
- 抓不到的数据留空
- 不允许删列

### 第8步：浏览器兜底（如需要）

对第6步结果中抓取状态为 `PARTIAL` 或 `POOR`，或`是否需要浏览器兜底 = 是`的条目：

调用 `detail-page-capture` 技能处理

兜底输出文件命名：
`YYYYMMDD_HHMMSS_{搜索关键词去域名}_浏览器兜底结果.xlsx`

## 关键路径

| 用途 | 路径 |
|------|------|
| 多页搜索脚本 | `~/Desktop/百度_Safari_H5抓取_多页.command` |
| 搜索结果 H5 目录 | `~/Desktop/h5/` |
| 处理结果目录 | `~/Desktop/处理文件夹/` |
| 政策详情输出 | `~/Desktop/detail-h5/` |
| H5 处理脚本 | `~/.openclaw/workspace-search/skills/search-h5-processor/scripts/process_search_h5_multi.py` |
| 原文抓取脚本 | `~/.openclaw/workspace-search/skills/detail-link-fetcher/scripts/fetch_detail_links.py` |

## 硬规则

1. 整套流程默认先走单结果扩词，并直接使用唯一结果继续执行
2. 第3步必须使用多页模拟点击百度搜索页的方式执行
3. 第3步保存完 H5 后必须关闭当前搜索页面，但保留 Safari 窗口
4. 第3步目标页数只是上限，实际抓多少页以页面可翻到的页数为准
5. 第4步结果必须覆盖上次执行结果
6. 第4步的原文链接优先输出真实政府落地链接
7. 第4步结果为 0 时，固定返回：`暂未搜索到相关内容`
8. 第4步结果非 0 时，自动继续进入第6步
9. 第6步默认优先采用直抓
10. 第8步浏览器兜底只处理缺失严重的条目，不默认全量再跑 Safari
