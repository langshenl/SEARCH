# Detail Page Capture

## Overview

这个技能只做一件事：

**接收已经确定好的原文链接，用 Safari 打开原文页，并把当前原文页面的完整 H5/HTML 保存到桌面专用文件夹。**

不要在这个技能里顺手做：
- 扩词
- 搜索结果页抓取
- 搜索结果解析
- 政策字段结构化抽取
- 去重清洗

先把“打开原文页并保存整页源码”这件事单独做好。

## Fixed Workflow

### 1. 接收原文链接

输入必须是已经从搜索结果页中确认好的原文链接。

默认规则：
- 不在本技能中自己找链接
- 不在本技能中自行搜索
- 不自动替换链接

### 2. 打开 Safari

使用 Safari 作为固定浏览器。

目标动作：
- 启动 Safari
- 打开给定的原文链接
- 等待页面完成基本加载

### 3. 获取当前原文页完整源码

核心目标：
- 获取当前原文页完整 H5 / HTML 页面内容
- 尽量保存渲染后的当前页面源码或等效 HTML 内容

### 4. 保存到桌面专用文件夹

输出目录固定为：
- `~/Desktop/detail-h5/`

要求：
- 如果文件夹不存在，则创建
- 每次任务保存为独立文件
- 文件名应包含：时间戳 + 链接主机名或安全标题

推荐文件名格式：
- `YYYYMMDD_HHMMSS_页面标识.html`

### 5. 输出说明

完成后返回给用户：
- 使用的原文链接
- 保存的文件完整路径
- 若失败，明确失败在哪一步（Safari 未打开 / 原文页未加载 / 页面源码未取到 / 文件写入失败）

## Hard Rules

- 这个技能当前只负责“打开原文页并保存整页 H5/HTML 文件”。
- 不在本技能中做政策字段抽取。
- 不在本技能中直接做政策清洗。
- 不要把“保存整页 HTML”和“结构化提取字段”混成同一步。

## Output Path

固定输出目录：
- `~/Desktop/detail-h5/`
