# Search H5 Processor

## Overview

这个技能只负责处理已经保存到桌面 `h5` 文件夹中的搜索结果页 HTML/H5 文件。

输出两类文件到桌面处理文件夹：
- 搜索结果 Excel
- 原文链接 md

每次执行必须覆盖上次结果，不做追加写入。

不要在这个技能里顺手做：
- 扩词
- 打开浏览器
- 再次搜索
- 打开原文链接
- 原文字段结构化抽取

## Fixed Workflow

### 1. 读取输入文件

默认从以下目录读取搜索结果页文件：
- `~/Desktop/h5/`

单页模式：优先处理最新的 `.html` / `.md` 文件；如用户指定某个文件，则按指定文件处理。
多页模式：按同一批次文件前缀或用户指定前缀，读取多页文件并合并解析。

### 2. 解析搜索结果页

从搜索结果页中尽量提取：
- 标题
- 简介
- 原始百度链接
- 原文链接（优先解析真实政府链接）

### 3. 输出到处理文件夹

固定输出目录：
- `~/Desktop/处理文件夹/`

默认输出文件：
- `搜索结果.xlsx`
- `原文链接.md`

其中 `搜索结果.xlsx` 必须只保留以下核心字段：
- 标题
- 简介
- 原始百度链接
- 原文链接

硬规则：
- 每次执行前，视同本次结果覆盖上次结果
- 不保留旧版 Excel / md 作为当前有效结果
- `原文链接.md` 优先写入真实政府落地链接；若确实无法解析出真实落地链接，再退回写原始百度跳转链接

### 4. 输出说明

完成后返回给用户：
- 处理的输入文件
- 提取出的结果条数
- Excel 路径
- md 路径

## Hard Rules

- 这个技能当前只负责“搜索结果页 → Excel + 原文链接 md”。
- 不在本技能中打开原文链接。
- 不在本技能中直接做政策详情抽取。
- 若页面本身是安全验证页或无有效结果页，应如实输出，而不是伪造结果。
