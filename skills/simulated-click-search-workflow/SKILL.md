# Multi-Page Simulated Click Search Workflow

## Overview

这是当前默认的“多页模拟点击搜索”主流程技能。

它只包含多页搜索相关流程：
- 单结果扩词
- 直接使用扩词返回的唯一结果
- Safari 打开百度并设置时间筛选
- 连续抓取多页搜索结果页 H5
- 关闭当前搜索页面但保留 Safari 窗口
- 合并处理多页 H5
- 输出搜索结果 Excel + 原文链接 md
- 结果非0时自动进入第六步
- 批量直抓原文链接
- 输出政策详情 Excel
- 标记需要浏览器兜底的条目

## Full Workflow

### 1. 输入原始搜索需求

用户输入自然语言需求，例如：
- 搜索湖北省政策
- 搜索河北省博物馆政策
- 搜索广西政策内容

### 2. 单结果扩词

先调用新的单结果扩词技能：
- `policy-single-query-expand`

规则：
- 只生成 **一个最终搜索词**
- 不再返回候选列表
- 不再等待用户选词
- 不再执行“默认选第1条”这一步
- 扩词结果直接进入后续多页搜索流程

例如：
- 搜索湖北省政策 → `2026年 湖北省政策 site:www.hubei.gov.cn`
- 搜索湖北省新能源相关政策 → `2026年 湖北省新能源政策 site:www.hubei.gov.cn`
- 搜索全国新能源政策 → `2026年 全国新能源政策 site:gov.cn`

### 3. 直接使用唯一扩词结果

当前整套流程的默认规则是：
- 自动使用单结果扩词技能返回的唯一结果进入后续步骤

### 4. 多页模拟点击百度搜索并设置时间筛选

使用桌面正式多页脚本：
- `~/Desktop/百度_Safari_H5抓取_多页.command`

脚本动作：
- Safari 打开百度
- 输入选中的搜索词
- 进入搜索结果页
- 点击“时间不限”
- 填入本年的第一天和最后一天
- 点击“确认”
- 保存第1页搜索结果 H5/HTML 到 `~/Desktop/h5/`
- 自动滚动到页码区
- 点击“下一页”或可用页码
- 继续保存后续页面 H5
- 到达目标页数上限或没有下一页时停止
- **保存完成后关闭当前搜索页面，但保留 Safari 窗口存在**

规则：
- 目标页数是上限，不保证一定抓满
- 如果实际页码不够，就抓到哪里算哪里
- 没有下一页时正常停止，不报错
- 结果是多少就是多少

脚本返回状态通常包括：
- 时间下拉状态
- 日期填写状态
- 每次翻页状态
- 实际保存页数
- 关闭页面状态
- 保存文件路径

### 5. 合并处理多页搜索结果 H5

调用：
- `search-h5-processor`

脚本：
- `~/.openclaw/workspace-search/skills/search-h5-processor/scripts/process_search_h5_multi.py`

输入：
- `~/Desktop/h5/` 中同一批次前缀的多页搜索结果文件

输出到：
- `~/Desktop/处理文件夹/`

固定输出文件：
- `搜索结果.xlsx`
- `原文链接.md`

规则：
- 覆盖上次结果
- 不做追加
- 合并多页结果后再去重
- `搜索结果.xlsx` 只保留以下字段：
  - 标题
  - 简介
  - 原始百度链接
  - 原文链接
- `原文链接.md` 优先写入真实政府落地链接

### 5.1 第5步结果为0时的固定处理

如果第5步提取结果数为 0：
- **直接返回：`暂未搜索到相关内容`**
- 不继续第六步

### 5.2 第5步结果非0时自动继续

如果第5步结果数大于 0：
- **第5步结束后立刻自动进入第六步**
- 不再停下来等待额外确认

### 6. 批量直抓原文链接

调用：
- `detail-link-fetcher`

脚本：
- `~/.openclaw/workspace-search/skills/detail-link-fetcher/scripts/fetch_detail_links.py`

输入：
- `~/Desktop/处理文件夹/原文链接.md`

规则：
- 串行抓取
- 每条间隔 1 秒
- 先对原文链接去重，再按唯一链接抓取
- 优先采用直抓，不默认打开 Safari

输出到：
- `~/Desktop/detail-h5/`

默认输出文件命名规则：
- `YYYYMMDD_HHMMSS_{搜索关键词去域名}_政策详情抓取结果.xlsx`

### 7. 政策详情输出字段

第六步输出的 Excel 中必须始终包含这些字段：
- 政策ID
- 标题
- 正文
- 摘要
- 发文机关
- 联合发文机关
- 发布时间
- 文号
- 政策层级
- 地区
- 所属行业
- 政策类型
- 政策主题
- 关键词
- 适用对象
- 支持方式
- 申报条件
- 申报时间
- 有效期
- 政策状态
- 原始链接
- 附件链接
- 相似政策
- 上位政策/下位政策关系
- 抓取状态
- 正文长度
- 是否检测到附件
- 是否需要浏览器兜底

规则：
- 抓不到的数据留空
- 不允许删列
- 第六步还要尽量给出完整度判断，便于后续兜底

### 8. 浏览器兜底策略

第六步与第七步输出文件命名统一采用：
- `YYYYMMDD_HHMMSS_{搜索关键词去域名}_...`

其中：
- 第六步建议命名为：`YYYYMMDD_HHMMSS_{搜索关键词去域名}_政策详情抓取结果.xlsx`
- 第七步建议命名为：`YYYYMMDD_HHMMSS_{搜索关键词去域名}_浏览器兜底结果.xlsx`

如果第六步结果中：
- 抓取状态为 `PARTIAL` 或 `POOR`
- 或 `是否需要浏览器兜底 = 是`

则这些条目属于第七步浏览器兜底候选。

当前兜底技能保留为：
- `detail-page-capture`

- 搜索结果处理目录：`~/Desktop/处理文件夹/`

## Hard Rules

- 这套流程是当前默认的“多页模拟点击搜索”流程。
- 整套流程默认先走单结果扩词，并直接使用唯一结果继续执行。
- 第四步必须使用多页模拟点击百度搜索页的方式执行。
- 第四步保存完 H5 后必须关闭当前搜索页面，但保留 Safari 窗口。
- 第四步目标页数只是上限，实际抓多少页以页面可翻到的页数为准。
- 第五步结果必须覆盖上次执行结果。
- 第五步的原文链接优先输出真实政府落地链接。
- 第五步结果为 0 时，固定返回：`暂未搜索到相关内容`。
- 第五步结果非 0 时，自动继续进入第六步。
- 第六步默认优先采用直抓。
- 第七步浏览器兜底只处理缺失严重的条目，不默认全量再跑 Safari。
